Execute Immediate in SnowSQL

A deep dive into using EXECUTE IMMEDIATE with SnowSQL and Snowsight.

Snowflake's EXECUTE IMMEDIATE command is a powerful feature that allows dynamic SQL execution. Understanding how to use it effectively can streamline your data workflows.

What is EXECUTE IMMEDIATE?

EXECUTE IMMEDIATE allows you to construct and execute SQL statements dynamically at runtime. This is particularly useful for scenarios where the SQL statement needs to be built programmatically based on runtime conditions.

Basic Syntax

EXECUTE IMMEDIATE $'''
  CREATE OR REPLACE TABLE dynamic_table AS
  SELECT * FROM source_table
  WHERE date_column = CURRENT_DATE();
''';

Use Cases

  • Dynamic Table Creation: Generate tables based on runtime parameters
  • Bulk Operations: Execute multiple DDL statements in a single call
  • Conditional Logic: Build SQL based on IF/ELSE conditions
  • Automation Scripts: Simplify repetitive administrative tasks

SnowSQL vs Snowsight

Both SnowSQL (command-line) and Snowsight (web UI) support EXECUTE IMMEDIATE. However, there are subtle differences in how variables and multi-statement scripts are handled between the two interfaces.
