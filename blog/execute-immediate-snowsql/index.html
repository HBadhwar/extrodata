Execute Immediate in SnowSQL

Dynamic SQL execution patterns for automation and conditional logic in Snowflake.

Snowflake's EXECUTE IMMEDIATE command enables dynamic SQL execution at runtime. It is a powerful tool for automation, conditional logic, and bulk operations — when used correctly.

What is EXECUTE IMMEDIATE?

EXECUTE IMMEDIATE constructs and executes SQL statements dynamically. This is useful when the SQL needs to be built programmatically based on runtime conditions, parameters, or metadata queries.

Basic Syntax

EXECUTE IMMEDIATE $'''
  CREATE OR REPLACE TABLE dynamic_table AS
  SELECT * FROM source_table
  WHERE date_column = CURRENT_DATE();
''';

Common Use Cases

  • Dynamic Table Creation: Generate tables based on runtime parameters or metadata
  • Bulk DDL Operations: Execute multiple schema changes in a single call
  • Conditional Logic: Build SQL branches based on IF/ELSE conditions evaluated at runtime
  • Automation Scripts: Simplify repetitive administrative tasks like role provisioning or warehouse management

SnowSQL vs Snowsight

Both SnowSQL (command-line) and Snowsight (web UI) support EXECUTE IMMEDIATE. There are subtle differences in how variables and multi-statement scripts are handled between the two interfaces — particularly around session state and variable scoping.
